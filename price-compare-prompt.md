# 提示词：AI 价格比价工具（单文件 HTML）

## 项目概述

为一个叫「竹昔」的用户做一个**纯前端、单文件 HTML** 的 AI 中转站价格比价工具。用户会在不同的 AI API 中转站（如云智AI、深蓝、DeepBIX等）之间比对同一个模型的价格，找出最便宜的渠道。

用户双击这个 HTML 文件就能在浏览器里用，不需要服务器、不需要安装任何东西，数据用 localStorage 保存在浏览器里。

---

## 数据模型

每一个"价格条目"包含以下字段，全部用 JSON 存储：

```javascript
{
  id: "唯一字符串（用 Date.now() + 随机数生成）",
  modelName: "模型名称，字符串，如 'gpt-4o-2024-08-06'",
  category: "自定义分类名，字符串，如 'GPT-4o 系列'",
  provider: "中转站名称，字符串，如 '云智AI'",
  useMultiplier: true/false,  // 是否启用倍率
  multiplier: number 或 null,  // 倍率数值，useMultiplier为false时存null
  baseInputPrice: number,      // 基础输入价格，单位：元/1M tokens
  baseCacheInputPrice: number, // 基础缓存输入价格，单位：元/1M tokens
  baseOutputPrice: number,     // 基础输出价格，单位：元/1M tokens
  createdAt: number,           // 录入时间戳
  updatedAt: number            // 最后修改时间戳
}
```

**实际价格的计算逻辑（用于排序和展示）：**
- 如果 `useMultiplier === true`：实际价格 = 基础价格 × 倍率
- 如果 `useMultiplier === false`：实际价格 = 基础价格（直接就是最终价格）

每一条记录有三个「实际价格」可以用于排序：
- 实际输入价格 = useMultiplier ? baseInputPrice * multiplier : baseInputPrice
- 实际缓存输入价格 = useMultiplier ? baseCacheInputPrice * multiplier : baseCacheInputPrice
- 实际输出价格 = useMultiplier ? baseOutputPrice * multiplier : baseOutputPrice

## 界面布局要求

### 整体结构（从上到下）

```
┌──────────────────────────────────────────────────────────────┐
│  顶部栏（固定在页面顶部，不随滚动消失）                      │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│  主内容区（可滚动）                                          │
│    - 分类卡片列表                                            │
│    - 每个分类卡片内是价格表格                                │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 顶部栏

从左到右排列：
- **标题**："AI 价格比对"（或者用户自定义的标题，大字号）
- **搜索框**：按模型名称或中转站名称模糊搜索，输入即过滤
- **「添加条目」按钮**：蓝色主按钮，最右边
- **「导出 JSON」按钮**：下载全部数据为 JSON 文件
- **「导入 JSON」按钮**：点击弹出文件选择框，读取 JSON 覆盖当前数据（导入前弹出确认对话框："导入将覆盖当前全部数据，确认？"）

### 主内容区 — 分类卡片

所有条目按 `category` 字段分组显示。没有条目的情况下显示一句话："还没有价格数据，点击右上角「添加条目」开始录入。"

**每个分类卡片：**

```
┌─ ▼/▶ GPT-4o 系列（3）  ──────── [⋮ 排序▼] [编辑分类] [删除分类] ─┐
│                                                                  │
│  ┌── 表格 ────────────────────────────────────────────────────┐  │
│  │ 模型名称 | 中转站 | 输入价 | 缓存输入价 | 输出价 | 操作    │  │
│  ├────────────────────────────────────────────────────────────┤  │
│  │ 每一行显示的是实际价格（即倍率计算后的价格）               │  │
│  │ 如果倍率启用，在价格后面用小字/灰色显示 "(×倍率值)"        │  │
│  │ 比如："60 (×2.0)" 或 "120 (×1.5)"                         │  │
│  │ 最右边操作列：[编辑] [删除]                                │  │
│  └────────────────────────────────────────────────────────────┘  │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

**分类卡片的关键行为：**
1. 点击分类标题可以折叠/展开（默认全部展开）
2. 卡片右上角有一个「排序」下拉按钮，三个选项：
   - 按输出价格↓（从高到低）
   - 按输入价格↓（从高到低）
   - 按缓存输入价格↓（从高到低）
   - 默认：按输出价格从高到低
3. 每个分类独立排序，切换分类A的排序不影响分类B
4. 「编辑分类」按钮：弹出一个输入框修改分类名，该分类下所有条目的 category 字段同步更新
5. 「删除分类」按钮：弹出确认框"删除此分类及其下所有条目？"，确认后删除全部相关条目
6. 如果某个分类下条目数为 0，该分类卡片自动消失或隐藏

**每个条目的操作列：**
- [编辑] 按钮：弹出编辑弹窗，同添加弹窗（预填现有数据）
- [删除] 按钮：弹出确认框"删除这条记录？"，确认后删除

## 弹窗：添加/编辑条目

弹窗居中显示，背景半透明遮罩，点击遮罩空白处不关闭弹窗（只能点取消/保存关闭）。

### 弹窗内容（从上到下）：

1. **模型名称** — 文本输入框，placeholder: "如 gpt-4o-2024-08-06"
2. **分类** — 下拉选择框 + 旁边一个「+ 新建分类」小按钮
   - 下拉框里列出当前所有分类名，默认选第一个
   - 如果还没有任何分类，下拉框为空，需要用户先新建
   - 「+ 新建分类」按钮：弹出一个小输入框输入新分类名，确定后添加到下拉列表并自动选中
3. **中转站** — 文本输入框，placeholder: "如 云智AI"
4. **☑ 启用倍率** — 复选框（默认不勾选）
   - 不勾选时：倍率输入框隐藏或灰色不可用，下面的字段直接就是最终价格
   - 勾选时：倍率输入框出现，placeholder: "如 2.0"
5. **价格输入区**（三个输入框，单位标签为 "元/1M tokens"）：
   - 基础输入价格 — placeholder: "如 15"
   - 基础缓存输入价格 — placeholder: "如 7.5"
   - 基础输出价格 — placeholder: "如 60"
   - 标签文案随「启用倍率」状态变化：
     - 启用倍率时标签写「基础输入价格」
     - 不启用时标签直接写「输入价格」
6. **底部按钮**：[取消]（灰色）[保存]（蓝色）

### 弹窗交互细节：
- 勾选/取消「启用倍率」时，倍率输入框立即显示/隐藏，价格标签立即切换
- 任何字段为空点击保存时：用红色提示"请填写所有价格字段"（模型名、分类、中转站也必须填）
- 编辑弹窗打开时，所有字段预填当前值
- 保存后弹窗关闭，页面数据立即刷新

## 导出/导入

### 导出 JSON
点击「导出 JSON」按钮，触发浏览器下载。JSON 结构：

```json
{
  "exportedAt": "ISO时间戳字符串",
  "version": 1,
  "entries": [所有条目的数组]
}
```

文件名格式：`ai-price-data-YYYY-MM-DD.json`（YYYY-MM-DD 为当天日期）

### 导入 JSON
点击「导入 JSON」按钮，弹出文件选择框（`<input type="file" accept=".json">`）。

- 读取文件内容，校验 JSON 格式
- 如果 JSON 结构不符合（缺少 entries 数组或其他字段不匹配），弹出错误提示"文件格式不符，请选择正确的导出文件"
- 校验通过后弹出确认框："导入将覆盖当前全部 X 条数据，确认继续？"
- 确认后，用文件中的 entries 替换 localStorage，页面刷新
- 如果导入的文件中有新分类，分类正常创建

## 搜索功能

顶部搜索框输入文字时：
- 对所有条目进行模糊匹配，匹配字段：modelName（模型名称）、provider（中转站名称）
- 匹配到的条目所在的分类卡片保持可见
- 没有匹配到的条目暂时隐藏（不是删除，取消搜索后恢复）
- 搜索为空时显示全部
- 搜索框旁边有个「✕」清空按钮，点击清空搜索恢复全部显示

## 技术实现要求

### 必须是一个单独 HTML 文件
所有 HTML、CSS、JavaScript 写在一个文件里。

### 技术栈
- 纯 HTML5 + CSS3 + 原生 JavaScript（ES6+）
- **不要引入任何外部依赖**（不引用 jQuery、Bootstrap、React、Vue 等）
- 不要使用 CDN 链接

### CSS 要求
- 页面背景：浅灰色，类似于 #f5f5f5 或 #f0f2f5
- 卡片：白色背景，圆角 8px，轻微阴影（box-shadow）
- 字体：系统默认字体栈（-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif），不要引 Google Fonts
- 配色主色调：蓝色系，类似 #4A90D9 或 #2563EB
- 按钮：蓝色主按钮、灰色取消按钮
- 表格：斑马纹（交替行颜色），header 背景稍深
- 整体风格：干净、现代、不花哨
- 响应式：在桌面浏览器（1280px+ 宽度）上正常显示，在 768px 以上也能看（折叠部分列）
- 弹窗居中，遮罩层半透明黑色

### JavaScript 要求
- localStorage key 统一用 `'ai-price-compare-data'`
- 所有数据操作（增删改查）封装成函数，不要散落在各处
- 页面加载时从 localStorage 读取数据，如果没有数据则初始化为 `[]`
- 每次增删改后立即写入 localStorage
- 每次增删改后立即重新渲染页面（可以整页重新渲染，数据量不大）
- 一个核心渲染函数 `render()` 负责根据当前数据、搜索词、各分类排序方式画出整个页面
- 排序状态（每个分类当前按什么排）记住在页面状态里，不存 localStorage（刷新后恢复默认按输出排）
- 折叠/展开状态记住在页面状态里（同上）
- 删除分类时先统计该分类下条目数，确认框里写清楚"N个条目将被一并删除"
- 倍率启用/不启用的状态切换在弹窗里即时反映

### 安全/边界情况
- 所有用户输入的数字用 `parseFloat` 解析，非数字时提示用户
- 价格显示保留最多 2 位小数
- 导入 JSON 时做严格校验：必须是一个对象，必须有 `entries` 数组，数组中每个元素必须有全部必要字段
- 如果条目数量超过 500，顶部显示一个计数"共 X 条记录"（正常情况不会到 500，但要有这个上限意识）

## 验收标准

生成代码后，应该满足以下所有点：

1. 双击 HTML 文件在 Chrome/Edge 浏览器中正常打开
2. 可以添加一条新记录（填模型名、分类、中转站、价格，保存后在分类卡片表格中看到）
3. 可以编辑已有记录的所有字段，包括切换倍率状态
4. 可以删除一条记录
5. 可以删除整个分类
6. 可以编辑分类名，所有相关条目同步更新
7. 可以在一个分类内切换排序方式（输出/输入/缓存输入）
8. 导出 JSON 文件能正常下载
9. 导入刚才导出的 JSON 文件能恢复全部数据
10. 搜索功能能过滤模型名和中转站名
11. 倍率启用后，价格显示为基础价×倍率的计算结果，并标注 "(×倍率值)"
12. 关闭浏览器再打开，数据还在
13. 没有任何外部 CDN 依赖，断网也能用
