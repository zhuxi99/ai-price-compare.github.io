# AI 价格比对

线上地址：

- Surge：https://ai-price-compare.surge.sh
- GitHub Pages：https://zhuxi99.github.io/ai-price-compare.github.io/

## 本地批量抓取倍率

双击项目目录中的“AI价格比对－倍率抓取”。工具会自动从本地最新价格 JSON 提取站点名称和地址、按域名去重并保存；以后价格数据中出现新站点也会自动补入。你只需为需要鉴权的站点补一次访问令牌，以及部分 New API 分支需要的用户 ID。之后直接点击圆角站点框即可多选，也可以全选后一次批量抓取。

站点资料保存在 `~/.config/ai-price-compare/`。访问令牌使用本机 AES-256-GCM 密钥加密，配置文件与密钥文件权限均限制为当前用户；令牌不会进入价格 JSON、日志或线上网站。编辑已保存站点时，令牌留空会保留原值，也可以明确删除或替换。

每个站点可单独设置充值比例：绝大多数普通站点 1:1 填 `1`，只有充值 1 元到账 10 美元额度的特殊站点才填 `10`。工具统一保存美元基础价和一个“最终倍率”，计算方式为“最终倍率 = 站点分组倍率 ÷ 到账额度”；站点名称中已有 `1:12充值`、`1：15充值` 等字样时会自动识别。

抓取器会自动识别标准 New API、One Hub 和 Sub2API。New API 可填写个人资料中的访问令牌；Sub2API 站点需要填写网页登录 JWT（不是 `sk-` API Key），工具会自动读取该账号下的有效 API Key、模型列表和分组倍率，并使用 LiteLLM 官方价格表进行估算；JWT 失效后会单独提示更新。

批量抓取完成后，服务端只会保留名称恰好为 GPT 5.6 sol 与 Claude Fable 5 的模型；空格、大小写和连字符写法视为同名，`plus`、`pro`、`MAX`、括号和说明文字等额外后缀会排除。价格统一按 `$ / 1M tokens` 展示。保存时按同一站点的最终输入、缓存和输出价格比较：有变化才更新，未变化不会改时间；如果全部未变化，则不会生成新的价格 JSON。随后使用原来的“一键发布”即可更新网站。

## 更新并发布数据

1. 在网页中完成价格修改。
2. 点击“导出 JSON”，将备份保存到下载目录。
3. 双击桌面上的“AI价格比对－一键发布”，等待终端显示 Surge 和 GitHub Pages 均发布成功。

也可以在本项目目录运行：

   ```bash
   npm run deploy:data
   ```

脚本会从 `~/下载`（或 `~/Downloads`）自动选择最新的有效非空备份，将数据写入 `.surge/index.html` 和 GitHub Pages 使用的根目录 `index.html`，部署到固定 Surge 域名，然后自动提交并推送 `index.html` 到 GitHub `main` 分支。空备份和格式错误的文件会被跳过。

GitHub 推送遇到临时网络错误时会自动重试 3 次，每次超过 45 秒会终止挂起连接后重试。若仍失败，可查看项目目录中的 `一键发布.log`；再次双击发布会继续推送已经生成的本地提交，不会丢失数据。

也可以明确指定备份文件：

```bash
npm run deploy:data -- "/完整路径/ai-price-data-日期.json"
```

只生成 `.surge/index.html` 而不发布：

```bash
npm run deploy:prepare
```

如果 Surge 不在当前 PATH 中，可以指定可执行文件路径：

```bash
SURGE_BIN="/完整路径/surge" npm run deploy:data
```

每次部署新的导出文件后，访客刷新页面会自动切换到最新发布的数据。访客在自己浏览器中的临时修改不会上传到网站。
